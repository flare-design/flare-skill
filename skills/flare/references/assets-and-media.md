# Assets And Media

Use this for asset listing, uploads, public URL imports, and media insertion.

Dynamic source of truth:

```text
list_media_capabilities
```

## Asset Kinds

Account assets can be:

- `image`
- `video`
- `audio`

`list_assets` filters:

- `kind`: `all`, `image`, `video`, or `audio`
- `sourceType`: `all`, `upload`, or `generation`
- `limit`: 1-50

## Local Agent Images

For local image files generated by an agent:

1. Call `create_image_upload_session`.
2. POST raw file bytes to `uploadUrl` with `Authorization: Bearer <uploadToken>`.
3. Include `Content-Type` and `x-flare-file-size`.
4. Use the returned `assetId` with `insert_asset_image`.

MCP image upload limits:

- Image only.
- 20 MB max.
- Upload token expires after 10 minutes.
- Do not search for or expose the MCP OAuth access token.

## Public HTTPS Images

For public image URLs:

- Use `save_image_asset_from_url` when the user wants to save to Assets.
- Use `insert_agent_generated_image` when the user wants a public URL saved and placed on the canvas.
- Do not use data URLs or base64 payloads in MCP JSON.

## Existing Assets

- Use `insert_asset_image` for images.
- Use `insert_asset_video` for videos.
- Use `add_audio_track` for audio timeline tracks.

Video insertion supports `autoplay`, `loop`, `muted`, and `mediaFit`.

Audio tracks support `startMs`, `endMs`, `trimInMs`, `trimOutMs`, `volume`, `muted`, `fadeInMs`, `fadeOutMs`, and `duckUnderVoice`.

Manual captions use `add_caption_track` with `items` containing `startMs`, `endMs`, and `text`.

## General Media Uploads

The normal media upload path supports image, video, and audio up to 250 MB, subject to workspace storage quota. The MCP upload-session path is intentionally narrower and scoped to local agent image upload.
